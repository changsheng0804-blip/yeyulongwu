# 一夜鱼龙舞 · 咒成 — 概念设计方案

> 「打字，便有魔法回应。」  
> 不是播放动画，是你施咒，粒子世界变形。

## 一、定位

**当前 V5**：粒子聚合成文字 → "这是一个技术演示"  
**咒成版**：打出 "炎咒"，世界燃起来 → "这是一个体验"

区别不在代码，在输入与输出的因果性。

## 二、咒语体系

### 五系咒法 + 一系禁忌

| 咒系 | 元素 | 关键词 | 粒子行为 | 色相 | 音效 |
|------|------|--------|---------|------|------|
| 🜁 火 · 烈 | Fire | 裂、炎、焚、爆、灼、烬、赤、火、烧、焰 | 从中心猛烈弹射，边缘粒子不断崩裂飞散 | 0°–40° 红金 | noise burst 3kHz,0.15s |
| 🜃 土 · 镇 | Earth | 盾、壁、镇、固、磐、山、界、御、守 | 向内塌缩成球体→缓慢膨胀为旋转护盾 | 40°–70° 琥珀 | 低频 42Hz,0.3s rumble |

## 三、咒语识别

**不是 NLP。关键词命中 + 优先级即可。**

```
function classifySpell(text) {
  const rules = [
    {family:'forbidden', keywords:['灭','无','空','寂','终','虚','零'], priority:10},
    {family:'thunder',   keywords:['雷','霆','电','闪','亟','碎','霹','雳'], priority:8},
    {family:'fire',      keywords:['裂','炎','焚','爆','灼','烬','赤','火','烧','焰'], priority:6},
    {family:'earth',     keywords:['盾','壁','镇','固','磐','山','界','御','守'], priority:5},
    {family:'water',     keywords:['愈','生','春','息','润','流','清','疗','复'], priority:4},
    {family:'wind',      keywords:['幻','梦','迷','隐','镜','影','虚','花','雾'], priority:3},
  ];
  // 匹配最长关键词，未命中默认 wind
}
```

### 边界情况
- 无关键词命中 → 默认走"风·幻"系（最中性的效果）
- 多系关键词 → 选优先级更高的
- 纯英文 → 同样的关键词映射（fire/water/thunder etc.）

## 四、音乐架构：基态 + 咒语调质

**V5 的问题：预编四轨总谱，换风格就是换总谱。咒成版的核心区别：音乐不是预编的，是被咒语调制的。**

### 基态音乐（没有咒语时）

持续运行的 ambient 层：
- 低密度底鼓（2拍1次）
- 稀疏踩镲（4拍1次）
- 持续和弦垫（三角波，极低音量）
- **无旋律线** — 旋律由咒语带来

### 施咒过程的音乐弧线

```
基态 ──┬── 聚合期(0→1.5s) ──┬── 咒成瞬间(1.5s) ──┬── 持续期(1.5→6s) ──┬── 消散(6s→) ── 基态
       │                      │                    │                    │
  鼓密度线性递增       触发咒系专属音效burst    音乐进入咒语调质模变    回归基态
  和声垫音量渐大       粒子成形那一刻                  │
                       是音乐的最强瞬态            火: 打击乐密度×2 + noise层
                                                   水: 琶音浮现 + 和弦垫音量↑
                                                   风: delay feedback↑ + 混响↑
                                                   雷: 隔拍极低频脉冲
                                                   土: 贝斯音量+50%, 旋律减半
                                                   禁忌: 全轨道静默
```

### 音效 burst 规格

| 咒系 | 实现 | 参数 |
|------|------|------|
| 火 | noise → bandpass | center=3000Hz, Q=0.5, decay=0.15s |
| 水 | 3×triangle osc 泛音 | freqs above root, soft attack 0.25s |
| 风 | stereo delay | L/R phase offset 0.13s, feedback 0.6 |
| 雷 | sawtooth sweep | 500→8000Hz, 0.08s |
| 土 | sine 低频 | 42Hz, 0.3s, 轻微 pitch 下滑 |
| 禁忌 | 全轨道 instant mute | → 2s silence → 30Hz fade in |

## 五、咒语后效动画系统

**每种咒系的文字成形后，粒子不是"站着的文字"，而是处于后效状态。**

### 火系 · 燃烧后效

```
文字成形后：
- 边缘 15% 粒子被赋予向外的随机初速度
- 每帧补充：从文字表面随机选 3% 粒子赋予新速度
- 这些粒子颜色锁定在 10°–35°（红→橙）
- 远离文字 1.5 单位后减速→消失→从文字中心重生
→ 视觉：文字在燃烧，火星不断迸发又回来
```

实现：
```js
function fireAftereffect(i, N) {
  if (Math.random() < 0.03 || (vx[i]==0 && vy[i]==0 && vz[i]==0)) {
    const dir = randomUnitVector();
    vx[i] = dir.x * (0.8 + Math.random()*2);
    vy[i] = dir.y * (0.8 + Math.random()*2);
    vz[i] = dir.z * (0.8 + Math.random()*2);
  }
  // 远离锚点后回拉
  const dist = distance(px[i],py[i],pz[i], anchorX, anchorY, anchorZ);
  if (dist > 1.5) {
    // 瞬间传送回文字中心附近
    px[i] = anchorX + smallNoise();
    py[i] = anchorY + smallNoise();
    pz[i] = anchorZ + smallNoise();
  }
}
```

### 土系 · 护盾后效

```
文字成形后：
- 粒子目标改为球形表面分布（半径 1.8）
- 所有粒子绕球面缓慢旋转
- 文字位于球心
- 每隔 0.5s 全体粒子向外推 0.1 单位再弹回（护盾脉冲）
→ 视觉：文字被一个旋转的粒子球壳保护着
```

### 水系 · 蒸发后效

```
文字成形后：
- 粒子从文字表面以 0.05 单位/秒的速度缓慢剥离
- 剥离的粒子透明度（或距离）线性衰减
- 剥离粒子颜色从蓝渐变到浅蓝到白
- 文字本体粒子不断减少→从底部重新补充
→ 视觉：文字表面不断有水汽升腾消散，永不断绝
```

### 风系 · 镜像后效

```
文字成形后：
- 生成第二组锚点 = 原始锚点 × 镜像矩阵（沿 YZ 平面对称）
- 两组锚点交替作为目标：每隔 0.3s 切换
- 实际粒子在两个位置之间振荡（不是硬切换，是弹簧拉到新目标）
- 周期性 50% 粒子闪烁（透明度→0.3 再回来）
→ 视觉：文字在两个位置间飘忽不定，像海市蜃楼
```

### 雷系 · 闪链后效

```
文字成形（几乎是瞬间——聚合期仅 0.15s）：
- 成形后文字抖动：pz += sin(t*60)*0.03（高速微振）
- 每隔 0.2s：随机选择 2 个粒子 → 两点间所有粒子按直线排列→瞬间发白→恢复
→ 视觉：文字持续抖动，偶有闪电链穿过文字内部
```

### 禁忌 · 归零后效

```
成形瞬间：
- 所有粒子以 3 倍弹簧速度向原点塌缩
- 画面在 0.5s 内完全变黑（粒子全部归零点）
- 保持全黑 2s
- 从原点爆发出新的粒子云（宇宙大爆炸般）
- 新粒子随机归位成文字
→ 视觉：世界毁灭→重生
```

## 六、交互流程

### 输入框 → 粒子空间 的咒语飞入

```
Enter 施咒：
1. 输入框文字闪白 0.1s
2. 文字内容以 CSS transform 从输入框位置飞向画布中心
   动画：scale(1→3) + opacity(1→0) + 0.4s ease-in
3. 飞到画布中心时——粒子开始聚合
4. 右上角浮现咒系符文（Canvas 绘制几何符号，0.3s fade in）
5. 左下角咒语历史更新
```

### 咒语历史

```
左下角：
┌─────────────┐
│ 炎咒  △     │  ← 最近
│ 雷霆  ⚡     │
│ 治愈  ○     │
└─────────────┘
  卷轴式展开，最多 3 条
  每条包含：咒语文字 + 咒系符文 + 渐变淡出
```

### 咒语叠加规则

- 新咒语触发 → 当前后效立即中断 → 粒子回公式流 → 开始新聚合
- 不做咒语叠加（会导致效果混乱）
- "咒语冷却"：上一次施咒的持续期未结束时，新咒语可以打断（这本身就是一种玩法）

## 七、技术架构

### 复用 V5 模块
- ✅ 粒子弹簧物理引擎（Float32Array + springK + damp）
- ✅ 3D 透视渲染管线（旋转矩阵 + fov）
- ✅ Canvas 文字采样 → 3D 锚点（1024×256 + 阈值 190）
- ✅ 音频引擎基础（oscillator/filter/gain/delay/noise）
- ✅ 状态机框架（HS_IDLE/APPEAR/HOLD/DISSOLVE → SPELL_CHARGING/CAST/EFFECT/FADE）
- ✅ 字体兜底链（@font-face CJKFallback + FONT_STACK）

### 新增模块

| 模块 | 代码量估算 | 说明 |
|------|----------|------|
| `classifySpell()` | ~30 行 | 关键词查表 |
| `spellEffectBurst()` | ~60 行 | 6 种音效 |
| `spellAftereffect_*()` | ~150 行 | 6 种后效函数 |
| `ambientMusicLoop()` | ~80 行 | 基态音乐替代预编总谱 |
| `spellModulateMusic()` | ~60 行 | 咒语调质参数 |
| `drawRune()` | ~30 行 | 符文绘制 |
| 输入框动画 + 飞入 | ~40 行 | CSS + JS |
| 咒语历史 UI | ~50 行 | Canvas 叠加层 |
| **合计新增** | **~500 行** | 在 V5 的 ~550 行核心代码上 |

### 不做的事
- ❌ 不做 NLP / 语义理解（关键词足够）
- ❌ 不做组合咒语（v1 单咒语）
- ❌ 不做符纸背景纹理（CSS 渐变即可）
- ❌ 不做墨迹粒子（太重，性能不够）

## 九、与 V5 的关键差异对比

| 维度 | V5 粒子成形 | 咒成版 |
|------|-----------|--------|
| 输入含义 | 随便打字 | 打字 = 施咒，有魔法后果 |
| 粒子行为 | 全系统一（聚合→定形→溶散） | 每系不同后效（燃烧/护盾/蒸发/闪烁/闪电/归零） |
| 音乐 | 预编四轨总谱，固定风格 | 基态 ambient + 咒语调质 |
| 色彩 | 彩虹色循环 | 色相锁定在咒系区间 |
| 独特性 | "又一个好看的粒子可视化" | "打字有魔法回应" |
| 分享价值 | 低（打开看一次） | 高（"输入'雷霆'试试"） |
