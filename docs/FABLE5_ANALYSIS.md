# Fable 5 乐章版 详细分析

## 核心创新：从"生成"到"作曲"

V2 的逻辑是**图形状态自下而上驱动音乐生成**——粒子跑到哪，音乐跟到哪。
Fable 5 倒过来：**一份手工编排的四轨总谱自上而下驱动图形**——每个十六分音符都定好了位置，图形充当音乐的视觉翻译官。

## 关键设计

### 1. 字符式记谱法
一维字符串代表 16 分音符网格，`x`/`R`/数字/`-` 四种符号覆盖全曲，极度精简且直接可编辑。

### 2. 音画在音频时钟上对齐
视觉事件和音频事件挂在同一根 `AC.currentTime` 时间线上，不依赖 setTimeout 或帧计数，同步精度达到音频采样级。

### 3. 三种鼓的差异化视觉角色
- 底鼓 → 粒子膨胀 + 镜头微震（camDist × 0.972）
- 军鼓 → 粒子抖动 + 亮度暴涨 + 色相偏移
- 踩镲 → 微弱亮度，高频轻盈感

### 4. 四乐章结构
星涡序曲 → 鱼龙行板 → 环面急板 → 球谐辉煌
每个乐章独立配置：图案 + 物理参数 + 音色 + 旋律密度 + 光迹长度

### 5. 视觉编排
光迹长度随乐章递减（8→4→3→2），慢乐章长拖尾制造流动感，快乐章短拖尾强化节奏颗粒感。

## 与 V2 的对比

| 维度 | V2 (蒙特卡洛版) | Fable 5 (乐章版) |
|------|----------------|-------------------|
| 音乐来源 | 图形实时生成 | 手工编排总谱 |
| 结构 | 无限循环、临界态探索 | 四乐章起承转合 |
| 节奏 | 由粒子动能决定 | 精确 16 分音符网格 |
| 形态控制 | 手动/自动变形 | 每个旋律音牵动 x |
| 鼓声部 | 无 | 三件套全配齐 |
| 核心体验 | 陪看/沉浸/不确定性 | 演出/观看/确定性 |
| 可预测性 | 低（每次播放不同） | 高（相同总谱） |
| 类比 | 爵士即兴 | 交响乐总谱 |
