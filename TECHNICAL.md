## V1 → V2 改进思路

### V1 的局限

V1 意境版是一个纯 2D 粒子动画——通过参数方程在 Canvas 上绘制带有 HSL 渐变光晕的点阵。它的视觉效果不错，但存在几个天花板：

1. **空间维度受限**：所有计算和渲染都在 2D 平面，缺乏深度感和立体空间表达
2. **无交互反馈**：用户只能调滑块，不能直接操作画面
3. **纯视觉单向**：图形不会回应任何外部刺激
4. **图案单一性**：只有一套公式通过 x 参数变形，变化空间有限

### V2 的设计目标

V2 的核心设计意图是**让图形和音乐互为因果**——不是"给动画配乐"，而是动画的物理状态本身驱动音乐生成，音乐的节拍再反过来穿透粒子运动。

为此引入了四个层面的改造：

#### 1. 二维 → 三维透视渲染

- 粒子坐标升级为 `(x, y, z)` 三元组
- 实现全景摄像机：yaw（水平旋转）/ pitch（俯仰）/ camDist（距离）可拖拽调节
- 透视投影公式 `sx = cx + x' * fov/z` 做深度缩放
- 粒子大小、亮度随 z 坐标动态调整

#### 2. 新增四套立体图案预设

| 预设 | 公式特征 | 视觉联想 |
|------|---------|---------|
| 鱼龙舞 | 多重正弦叠加 + 径向随机微扰 | 中国传统灯会 / 鱼龙穿梭 |
| 球谐光雕 | 离散球谐函数 `Y(l,m)` 模拟 | 原子轨道 / 量子概率云 |
| 环面绸带 | 环面参数方程 + 波动变形 | 拓扑雕塑 / 莫比乌斯带 |
| 星涡 | 对数螺旋 + 指数衰减径向密度 | 银河旋臂 / 台风眼 |

#### 3. 粒子物理模拟

V1 每帧直接计算目标位置，粒子"瞬移"到目标。V2 改为：

- **弹簧力**：粒子被拉向公式计算的目标位置 `F = -k(x - target)`
- **漩涡力**：y 轴上的旋转切向力，制造环绕流动感
- **阻尼衰减**：`Math.exp(-4.2 * sdt)`，帧率无关的指数阻尼
- **冲击波**：点击画面时，所有粒子从中心向外爆裂

这使得粒子有了"质量"——它们追赶目标轨迹时会有惯性、超调、回摆，视觉上更有"液态光"的质感。

#### 4. 音画互驱引擎

这是 V2 最核心的创新。设计了一条完整的闭环：

```
┌──────────────────────────────────────────────┐
│                                              │
│  形态参数 x ────→ 旋律音高走向（级进旋律）      │
│  粒子动能 ──────→ 节奏密度 + 音色明暗          │
│  粒子聚散度 ────→ 音符音量                    │
│  图案切换 ──────→ 调性转调 + 音色切换          │
│  画面点击 ──────→ 冲击波 + 锣声 + 下行音阶     │
│                                              │
│  音乐节拍 ──────→ 反向驱动粒子视觉脉冲          │
│                                              │
└──────────────────────────────────────────────┘
```

四套图案各有独立的调性档案：

| 图案 | 调式 | 主音 (MIDI) | 音色 | 低音进行 |
|------|------|------------|------|---------|
| 鱼龙舞 | A 羽调 | 57 | Triangle（弹拨感） | 0→3→8→10 |
| 球谐光雕 | C 宫调 | 60 | Sine（铃音） | 0→-3→5→7 |
| 环面绸带 | D 商调 | 62 | Sawtooth（暖弦） | 0→-2→3→5 |
| 星涡 | E 羽低音 | 52 | Sine（慢板） | 0→-4→3→-2 |

## 代码架构说明

### 整体流程

```
初始化
  ├─ 创建 Float32Array 存储 16000 粒子 (位置+速度+随机种子)
  ├─ 初始化 Web Audio Context (延迟加载)
  └─ 绑定 UI 事件

主循环 (requestAnimationFrame × 60fps)
  ├─ 计算当前形态参数 (自动变形 / 手动)
  ├─ 物理步进
  │   ├─ target() → 算目标位置
  │   ├─ 弹簧力 + 漩涡力 + 阻尼
  │   ├─ 更新位置 / 速度
  │   └─ 抽样统计动能 & 聚散 → 喂给音乐
  ├─ 节拍脉冲包络 (音乐→视觉反向驱动)
  ├─ 渲染
  │   ├─ 半透明黑色覆盖 (光迹拖尾)
  │   ├─ 透视投影
  │   ├─ HSL 颜色计算 + 缓存复用
  │   └─ fillRect 像素绘制
  └─ requestAnimationFrame → 下一帧

音频调度器 (setInterval × 40Hz)
  ├─ 读取当前图形状态统计量
  ├─ 前瞻排程 (提前 0.12s)
  └─ 生成音符 → Web Audio API
```

### 关键模块

#### 1. 内存布局 — Float32Array

```js
const px = new Float32Array(NMAX);  // x 坐标
const py = new Float32Array(NMAX);  // y 坐标
const pz = new Float32Array(NMAX);  // z 坐标
const vx = new Float32Array(NMAX);  // x 速度
const vy = new Float32Array(NMAX);  // y 速度
const vz = new Float32Array(NMAX);  // z 速度
```

用 7 个连续的 Float32Array 替代对象数组，这样做的好处：

- 内存布局连续，CPU 缓存友好，遍历时预取命中率高
- 避免万级对象的堆分配和 GC 压力
- 粒子数 10000 时已经显著优于 `{x, y, z, vx, vy, vz}` 的对象写法

#### 2. 帧率无关物理

```js
const damp = Math.exp(-4.2 * sdt);
```

阻尼力用 `exp(-k * dt)` 而非 `speed *= 0.95`。这样无论设备跑 30fps 还是 120fps，粒子的衰减曲线完全一致。`sdt` 是帧间隔 × 动画速度，`dtms` 被 `Math.min(now-last, 50)` 硬截断，防止切标签页回来后的大跳变。

#### 3. 颜色缓存

```js
const colorCache = Object.create(null);
function col(h, l) {
  h = ((h | 0) % 360 + 360) % 360; h -= h % 4;  // 量化到 4° 阶梯
  l = l | 0; l -= l % 5;                          // 量化到 5% 阶梯
  const k = h * 100 + l;
  let c = colorCache[k];
  if (!c) { c = 'hsla(' + h + ',100%,' + l + '%,0.9)'; colorCache[k] = c; }
  return c;
}
```

`fillStyle = 'hsla(...)'` 涉及浏览器内部的字符串解析和颜色空间转换，是 Canvas 2D 渲染的热路径。通过量化 hue 到 4°、lightness 到 5% 的阶梯，同名颜色直接复用缓存的字符串，避免每帧为 10000+ 粒子重新构造颜色。

#### 4. 音频前瞻排程

```js
setInterval(() => {
  while (nextNoteTime < AC.currentTime + 0.12) {
    scheduleStep(step, nextNoteTime, spb);
    nextNoteTime += spb;
    step = (step + 1) % 32;
  }
}, 25);
```

核心思路是利用 Web Audio 的精确时间线：音符不是在"现在"播放，而是排到未来 0.12 秒的时间点上。这样即使 `setInterval` 有 5-10ms 的抖动，音符之间相对间隔依然准确。

额外处理了 `document.hidden` 场景——页面切到后台时浏览器会把定时器降到每秒 1 次的极低频，此时如果硬要继续排音符会积压大量过时事件。代码直接重置 nextNoteTime，回到前台后从当前时间重新开始，避免音序混乱。

#### 5. 图形→音乐的映射

旋律走向由形态参数 x 决定：

```js
const n01 = (curMorphX + 60) / 120;      // 归一化到 [0,1]
const degTarget = Math.round(n01 * 9);    // 映射到 10 级音阶
```

音符不是直接跳到目标音级，而是**级进**（每次最多移动 1 阶），避免跳进的生硬感。同时用 `probeOffset()` 取一个探针粒子的空间高度对音高做 ±2 的微调，让旋律能微妙地回应三维形态的起伏。

- `sEnergy`（粒子动能）→ 音符密度、filter cutoff（音色明暗）
- `sSpread`（粒子聚散度）→ 音符音量
- `AC.currentTime - lastBeatT` → 节拍包络，反向驱动粒子大小和亮度脉动

#### 6. 冲击波与锣声

点击画面触发：

1. `shockwave()` 对所有粒子施加从中心向外的瞬时冲量 `~ 1/r²`，加上少量随机抖动
2. `gong()` 用白噪声 buffer 通过带通滤波器频率扫描模拟锣声（950Hz→160Hz），叠加低频正弦震动和下行五声音阶炸裂

## 技术栈

- **HTML5 Canvas 2D**：像素级粒子绘制 + `globalCompositeOperation: "lighter"` 叠加混合
- **Web Audio API**：振荡器合成、滤波器、延迟效果、动态压缩器
- **原生 JavaScript**：零外部依赖，单文件交付
